Highlight #1 Version Control

Version control is an extremely efficient way to create or edit files. It allows users to revert to previous versions of a file to either edit, or simply recall for convenience. It lets the user code with ease, letting them know that every state or version of the files they are working on will be saved in case of an unexpected emergency. You can also compare your changes, revert the entire project back to a previous state, and much more. There are several types of Version Control Systems (VCS), including the following: local VCS's, centralized VCS's and distributed VCS's.

Highlight #2 GIT

GIT is a type of VCS, but it is distinctly different from other versions. One way it is different is how it stores data. GIT saves 'snapshots' of the entire file and stores it into its own miniature file system, instead of simply saving the file. Each time you save the state. it goes through this process, which is very efficient. Another important distinction is that GIT does not always require files outside of the local system --in other words, almost every GIT operation is local. No other computer or network is required, and it does this by reading from your local database, which in result is much faster because GIT does not have to go out to the server to reach the file history.

Highlight #3 GIT Repository

A GIT repository is where all of your files, states, and edits are located in. The repository contains a set of commit objects, reference to commit objects called 'heads,' among other things. A commit object contains the following: a set of files reflecting the projects previous states, parent commit object references, and a 40 character string that identifies the commit object called an SHA1 name. A head is a reference to a commit object, and each head has a specific name. The project you work on will be stored in the same location as the GIT repository, inside a subrepository '.git'.

Highlight #4 Basic GIT Commands

To fully take advantage of GIT, one must know the appropriate commands. Some of the most frequently used commands are: push, commit, status, add files, and create a new local repository. Push: sending changes to the master branch of the remote repository ('git push origin master'). Commit: changes files that one has previously added with 'git add', and commits any files that have changed since ('git commit -a'). Status: shows the user changed files, as well as those that still need to either commit or add ('git status'). Add files: adds one or more files to the index ('git add "filename", git add *'). Create a new local repository: ('git init').

Highlight #5 Remote Repositories vs Local Repositories

A remote repository is one that is shared among multiple users on a remote server, while a local repository is one that is not shared and resides locally for a single user. Users can create local repositories in the following ways: by either cloning an existing repository onto your own local machine, or by creating an entirely new repository. The great thing about a local repository is that you can edit, commit, and more without an internet connection, and once you reestablish a connection you can push your work to the remote repository. It makes working on your files much more simple and efficient.
